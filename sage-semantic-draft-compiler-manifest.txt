OLD PARALLEL CONTRACT  →  SEMANTIC-DRAFT CONTRACT
------------------------------------------------
 model emitted: criteria[] (strings), evidenceRequirements[] (strings), groundingV1{criteria[{criterionIndex,
   evidenceIndex,criterionKind,factRefs,transitionRef,pageUrl,stateId,evidenceMode,supportRationale}]},
   targetSurface, sources[], anchors[], verificationMethod   → many chances to mis-align indexes/bijection.
 model NOW emits: criteria[{text,evidenceRequirement,criterionKind,factRefs,transitionRef,evidenceMode,
   supportRationale}] — each criterion ONCE; NOTHING indexed or derived.

DETERMINISTIC DERIVATION TABLE (every removed model field)
----------------------------------------------------------
 criterionIndex        = i (the criterion's position)
 evidenceIndex         = i (always equals its criterion index)
 criteria[i]           = criterion.text
 evidenceRequirements[i]= criterion.evidenceRequirement
 pageUrl / stateId     state/content/visual → first cited fact; action_outcome → the cited AFTER-state fact
                         of a positively-safe transition (else reject)
 targetSurface         action first criterion → transition.startUrl; else first criterion's derived pageUrl
 sources[]             unique page refs from cited facts + transitions; observation = bounded ACTUAL observed text
 anchors[]             one verbatim per criterion (elementName ≥3 chars, else visibleText), deduped
 verificationMethod    bounded phrase from the criteria's evidence modes
 groundingV1           assembled from all of the above + the set digest

COMPILER REJECTION-CODE MATRIX (per-mission drop; bounded counts recorded)
--------------------------------------------------------------------------
 architect_fact_not_presented        a factRef not in the bounded observation VIEW shown to the architect
 architect_transition_not_presented  a transitionRef not in that view
 transition_not_safe                 an action_outcome cites a non-safe (state_changing/unverified) transition
 action_missing_after_state_fact     an action_outcome cites no fact from the transition's after-state
 derived_anchor_unavailable          no ≥3-char elementName/visibleText to anchor on
 (forbidden keys / bad enums / empty factRefs → rejected earlier by the strict transport Zod → schema_invalid)
